# 2026 CV Competition — детекция объектов (Kaggle)

Соревнование: **`2026-cv-competition`** на Kaggle.

## Задача

- **Тип:** object detection на **52 класса**.
- **Данные:** относительно небольшой датасет (ограниченный объём разметки/изображений).
- **Метрика:** **AP по всем классам при пороге IoU 0.5** (по смыслу — **mAP@0.5**).

## Структура данных (как в выгрузке)

В репозитории данные лежат в каталоге `2026-cv-competition/`:

| Часть | Путь |
|--------|------|
| Обучающие изображения | `2026-cv-competition/train/train/images/` |
| Разметка (YOLO txt) | `2026-cv-competition/train/train/labels/` |
| Тестовые изображения | `2026-cv-competition/test/test/images/` |
| Пример сабмита | `2026-cv-competition/sample_submission.csv` |

Файлы разметки в `labels/` — формат **YOLO**: одна строка на объект:  
`class_id x_center y_center width height` (все координаты **нормированы** относительно ширины/высоты изображения, значения в \([0,1]\)).

Имена файлов в тесте совпадают с колонкой `image_id` в `sample_submission.csv` (без суффикса `.jpg`).

## Формат сабмита (проверено на Kaggle)

Файл в корне решения (в требованиях курса): **`sample_submission.csv`**.

Колонки и важные детали:

- **`image_id`** — имя изображения **без расширения** (как в шаблоне Kaggle).
- **`PredictionString`** — строка из блоков по 6 значений на объект:  
  `class score x1 y1 x2 y2`
- Координаты должны быть в **пикселях** (не нормированные), с порядком  
  `x1 <= x2`, `y1 <= y2`.
- Пустые/`null` значения лучше не отправлять. Для строк без детекций используем безопасный плейсхолдер:  
  `0 0.000000 0 0 0 0`.

Если отправлять нормированные координаты или формат без `score`, валидатор может принять файл, но скор будет некорректным.

## Текущий зафиксированный baseline

- Лучший публичный скор на текущем пайплайне: **0.20769**.
- Рабочий режим инференса: `yolov8s_submit/best.pt`, `imgsz=704`, `augment=True`, постфильтр около `thr=0.12`.
- Финальный файл для загрузки: `sample_submission.csv` в корне репозитория.

## Правила честности и оформления (из условий курса / соревнования)

1. **Можно любые честные методы**, но **запрещено дополнять датасет** внешними данными / «левыми» картинками / сторонними датасетами для обучения. Аугментации **исходных** тренировочных изображений обычно допустимы, если это не противоречит тексту вашего задания на Kaggle.
2. Сабмиты должны быть **честными** (без неправомерного доступа к приватной разметке и т.п.).
3. **Антиплагиат:** попарное сравнение решений; копии — дисквалификация. Учитывайте, что шаблонный / ИИ-код может давать похожие решения.
4. **Сдача:** ссылка на **GitHub** с ноутбуком/кодом.
5. В ноутбуке должны быть явные разделы: **EDA**, **обучение модели**, **инференс**, **постобработка**. Итоговый артефакт — **`sample_submission.csv`**.
6. **Все случайные seed’ы зафиксировать на `993`** (`random`, `numpy`, `torch`, sklearn и т.д., везде где применимо).
7. **Идентификация:** ник на Kaggle и ФИО должны совпадать с требованиями курса (маппинг выдают отдельно).
8. **Тай-брейк:** при равенстве метрики выше тот, кто отправил **раньше**.
9. **Лидерборд:** есть public и **private**; итоговое место — по **private**. До конца соревнования нужно **выбрать**, какие сабмиты учитываются.

## Оценка в курсе (ориентир из условий)

- **Оценка 3 (до 55 баллов):** нужно **побить Baseline 1**; между Baseline 1 и Baseline 2 баллы **линейно**.
- **Оценка 4 (до 65 баллов):** **побить Baseline 2**.
- **Выше Baseline 2:** линейное распределение от **65 до 100** баллов.
- **Топ-3** (при условии, что они **выше Baseline 2**): возможны **доп. баллы к экзамену** (макс. **0.4** за экзамен, т.е. старт с **0.1**). Чтобы их получить: **нельзя** идти на **автомат**, нужна **презентация 5–7 минут** на паре с демонстрацией шагов **EDA, обучения и постпроцессинга**.
- Итоговое распределение оценок — **после** чистки лидерборда от нарушителей.
